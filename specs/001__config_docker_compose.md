## Objetivo
Criar um docker compose de dev e outro de produção.

## Pacote
Os arquivos devem ser criados na raiz do projeto.

## Referências
- Arquivo docker-compose.dev.yml
- O arquivo docker-compose.yml (produção)

## Explicações adicionais
- O arquivo docker-compose.dev.yml deve conter os serviços necessários para o ambiente de desenvolvimento, como banco de dados PostgreSQL.
- O arquivo docker-compose.yml (produção) deve conter os serviços necessários para o ambiente de produção, como o Traefik, além de configurações otimizadas para performance e segurança.
- Ambos os arquivos devem incluir volumes, redes e variáveis de ambiente conforme necessário.
